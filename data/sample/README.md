# Data

This project uses the schema of the **UK STATS19 road-safety dataset**,
published by the Department for Transport (DfT) under the
[Open Government Licence v3.0](https://www.nationalarchives.gov.uk/doc/open-government-licence/version/3/).

## What ships in this repo

Two **small, fully-synthetic** CSVs live in this folder so the pipeline runs
out of the box with no download:

| File | Rows (default) | Description |
|------|---------------|-------------|
| `collisions_sample.csv` | ~6,000 | one row per collision, STATS19 collision fields |
| `vehicles_sample.csv`   | ~11,000 | one row per vehicle involved |

These files contain **no real records**. They are generated by
[`src/generate_sample_data.py`](../../src/generate_sample_data.py) from random
distributions that mimic realistic STATS19 value ranges. Regenerate or resize
them at any time:

```bash
python src/generate_sample_data.py --rows 20000
```

## The `severity_score` target

Real STATS19 records carry a three-level `accident_severity`
(1 = Fatal, 2 = Serious, 3 = Slight). This project instead predicts a
**continuous severity index from 1 to 10**, engineered to turn the task into a
regression problem. In the sample generator the index rises with speed limit,
darkness, rural roads, poor weather/road surface, and the number of vehicles
and casualties, plus Gaussian noise. This is a modelling design choice, not a
field that exists in the raw data.

## Running on the real STATS19 data

1. Download the collision, vehicle and casualty CSVs from the DfT road-safety
   open-data page: <https://www.data.gov.uk/dataset/cb7ae6f0-4be6-4935-9277-47e5ce24a11f/road-safety-data>
   (or the DfT "Road Safety Data" landing page on data.gov.uk).
2. Place the large files under `data/raw/` — this folder is git-ignored, so
   they never get committed.
3. Point the loaders at them (adjust the paths in
   [`src/train_model.py`](../../src/train_model.py)) and, because the real file
   has `accident_severity` (1-3) rather than a 1-10 index, either map it to a
   score or switch the estimator to a classifier.

The full dataset is hundreds of MB and is intentionally excluded from version
control via [`.gitignore`](../../.gitignore).
